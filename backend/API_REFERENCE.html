Setup & Quick Start

🚀 Installation

1
Clone and install: npm install
2
Copy .env and fill in MONGO_URI, JWT_SECRET, Cloudinary keys
3
Start dev server: npm run dev
4
Health check: GET http://localhost:5000/health

🔑 Authentication

1
Register or login to get a JWT token
2
Add to all protected requests: Authorization: Bearer <token>
3
Token expires in 7 days

📤 File Uploads (Cloudinary)

1
Use multipart/form-data for any file upload endpoint
2
Max file size: 5MB. Formats: jpg, jpeg, png, webp
3
Item photos: max 4 files per upload (field name: photos)

Items

/api/items
POST /api/items 🔒 Auth Create new listing ▼
Request (multipart/form-data)
Field: title → "Old Laptop Dell i5" Field: description → "Working condition, minor scratches" Field: price → 5000 // 0 = FREE badge Field: category → "Electronics" Field: locationName→ "Kanpur, UP" // optional Field: lat → 26.4499 // optional override Field: lng → 80.3319 // optional override Files: photos → up to 4 image files
Categories
"Electronics" | "Furniture" | "Books" | "Clothes" | "Farm Tools" | "Other"
GET /api/items?lat=&lng=&radius= Get nearby items (geo) ▼
Query Params
ParamTypeRequiredDefaultDescription
latnumberRequired–User latitude
lngnumberRequired–User longitude
radiusnumberOptional5Radius in km (1–20)
pagenumberOptional1Page number
limitnumberOptional20Items per page
Example Request
GET /api/items?lat=26.4499&lng=80.3319&radius=5&page=1&limit=20
GET /api/items/my 🔒 Auth Get my listings ▼
Headers
Authorization: Bearer <token>
GET /api/items/:itemId Get single item detail ▼
Response 200

Search

All Query Params
ParamTypeRequiredDefaultDescription
qstringOptional–Keyword search (title + description)
latnumberOptional0User latitude
lngnumberOptional0User longitude
radiusnumberOptional5Search radius in km
categorystringOptional–Electronics / Furniture / Books / Clothes / Farm Tools / Other
minPricenumberOptional–Minimum price filter
maxPricenumberOptional–Maximum price filter
pagenumberOptional1Page number
limitnumberOptional20Results per page
Example Requests
// Search laptops within 10km of Kanpur GET /api/search?q=laptop&lat=26.4499&lng=80.3319&radius=10 // Electronics under ₹5000 GET /api/search?category=Electronics&maxPrice=5000&lat=26.4499&lng=80.3319 // Free items nearby GET /api/search?minPrice=0&maxPrice=0&lat=26.4499&lng=80.3319

Chat

/api/chat
POST /api/chat/conversations 🔒 Auth Start or get conversation for item ▼
Request Body
{ "itemId": "64f1a2b3c4d5e6f7a8b9c0d1" }
Response 201
{ "success": true, "data": { "_id": "conv_id...", "item": { "title": "Dell Laptop", "price": 5000 }, "buyer": { "name": "Rahul", "avatar": "..." }, "seller": { "name": "Priya", "phone": "98..." } } }
GET /api/chat/conversations 🔒 Auth List all my conversations ▼
Returns
Array of conversations sorted by last message date. Each includes: item info, buyer/seller names, lastMessage, unread counts.
GET /api/chat/conversations/:convId 🔒 Auth Get conversation details ▼
Note
Only the buyer and seller of this conversation can access it.
POST /api/chat/conversations/:convId/messages 🔒 Auth Send a message (REST fallback) ▼
Request Body
{ "text": "Is this still available?" }
Note
This also emits a new_message Socket.io event to the conversation room. Use Socket.io for real-time; this is a REST fallback.
GET /api/chat/conversations/:convId/messages 🔒 Auth Get all messages (paginated) ▼
Query Params
page=1 limit=50 // sorted oldest → newest
Side Effect
Marks all messages from the other person as isRead: true Resets your unread count to 0

Ratings

/api/ratings
POST /api/ratings 🔒 Auth Rate a seller after deal ▼
Request Body
{ "rateeId": "seller_user_id", "itemId": "item_id", "stars": 5, // 1 to 5 "comment": "Great seller, fast response!" // optional }
Note
Auto-recalculates seller's average rating after every new rating. One rating per (rater + item) pair — duplicate prevented.
GET /api/ratings/user/:userId Get all ratings for a user ▼
Response 200
{ "success": true, "count": 12, "data": [ { "stars": 5, "comment": "...", "rater": { "name": "Ankit" }, "item": { "title": "Dell Laptop" } } ] }

Reports

/api/reports
POST /api/reports 🔒 Auth Report a fake/spam listing ▼
Request Body
{ "itemId": "64f1a2b3c4d5e6f7a8b9c0d1", "reason": "This item is fake / already sold / spam" }
GET /api/reports 👑 Admin Get all reports (admin only) ▼
Query Params
status=pending // filter: pending | reviewed | resolved page=1 limit=20
To Make Admin
In MongoDB, manually set isAdmin: true on a user document. Then login with that user → their token will have admin access.
PATCH /api/reports/:reportId 👑 Admin Update report status ▼
Request Body
{ "status": "reviewed" } // pending | reviewed | resolved

Socket.io — Real-time Chat

ws://localhost:5000

🔌 Connection

// npm install socket.io-client import { io } from 'socket.io-client'; const socket = io('http://localhost:5000', { auth: { token: 'your_jwt_token_here' } // Required! }); socket.on('connect', () => console.log('Connected')); socket.on('connect_error', (e) => console.log('Auth error', e.message));
